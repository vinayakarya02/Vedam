# Deploy Vedam Events on Render

One-click style deploy using the included `render.yaml` blueprint.

## Database setup (before deploy)

Run in Supabase SQL Editor:

1. `supabase/migrations/001_initial_schema.sql`
2. Optional: `supabase/seed.sql`

## Quick deploy

1. Push this repo to **GitHub**
2. [Render Dashboard](https://dashboard.render.com) → **New** → **Blueprint**
3. Connect the repo — uses root `render.yaml` (deploys `backend/` + `frontend/`)
4. Set env vars for **vedam-events-api** and **vedam-events-web**
5. Click **Apply**

## After first deploy

| Service | Set |
|---------|-----|
| **vedam-events-api** | `FRONTEND_URL`, `APP_URL` = frontend URL |
| **vedam-events-web** | `NEXT_PUBLIC_API_URL` = API URL, `NEXT_PUBLIC_APP_URL` = frontend URL |

Redeploy both after updating URLs.

## Environment variables

| Variable | Required | Description |
|----------|----------|-------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Yes | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Yes | Supabase anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | Yes | Service role (server only) |
| `NEXT_PUBLIC_APP_URL` | Yes | Your Render URL (no trailing slash) |
| `RESEND_API_KEY` | Yes | Resend API key |
| `RESEND_FROM_EMAIL` | Yes | Verified sender email |
| `CRON_SECRET` | Auto | Generated by blueprint; used by cron job |
| `CORS_ALLOWED_ORIGINS` | Optional | Extra origins, comma-separated |

Render also sets **`RENDER_EXTERNAL_URL`** automatically — CORS allows it without extra config.

### Example CORS (optional)

Allow your marketing site to call the API:

```
CORS_ALLOWED_ORIGINS=https://vedam.org,https://www.vedam.org
```

## Supabase auth redirects

In Supabase → **Authentication** → **URL configuration**, add:

- **Site URL:** `https://your-app.onrender.com`
- **Redirect URLs:**
  - `https://your-app.onrender.com/**`
  - `http://localhost:3000/**` (local dev)

## Health check

Render uses: `GET /api/health`

```json
{ "status": "ok", "service": "vedam-events", "supabase": "configured" }
```

## Cron (email reminders)

The blueprint creates a **cron job** that hits `/api/cron/reminders` every hour.

Ensure `NEXT_PUBLIC_APP_URL` on the web service matches the live URL so the cron target is correct.

## Manual deploy (without blueprint)

1. **New Web Service** → connect repo
2. **Runtime:** Node
3. **Build command:** `npm ci && npm run build`
4. **Start command:** `npm run start:render`
5. **Health check path:** `/api/health`
6. Add all env vars from `.env.example`

## Local production test

```bash
npm run build
PORT=3000 npm run start:render
```

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Build fails on env | Add `NEXT_PUBLIC_*` vars in Render before deploy |
| Admin login redirect loop | Add Render URL to Supabase redirect URLs |
| CORS errors from another domain | Add domain to `CORS_ALLOWED_ORIGINS` |
| Cron emails not sending | Check cron logs; verify `CRON_SECRET` matches web service |
| Free tier sleeps | First request after idle may take 30–60s |
